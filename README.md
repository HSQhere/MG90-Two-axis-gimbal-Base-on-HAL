# MG90 双轴云台控制系统 (基于 STM32 HAL)

## 📖 项目简介
本项目实现了一个基于 STM32 微控制器的双轴伺服云台系统。系统具备高性能的实时响应能力，并支持两种工作模式：
* **摇杆控制模式**：通过电位器摇杆实时控制云台的 Yaw（偏航）和 Pitch（俯仰）角度。
* **MPU6050 自稳模式**：利用 MPU6050 六轴传感器数据，执行反向补偿算法，使云台在载体晃动时保持水平稳定。
* **无缝切换**：通过物理按键触发外部中断，实现两种模式的快速切换。

## 🛠️ 硬件连接

### 1. 核心外设引脚定义
| 外设功能 | 引脚名称 | STM32 引脚 | 备注 |
| :--- | :--- | :--- | :--- |
| **舵机 (Servo)** | Yaw (y) | PA0 | TIM2 Channel 1 (PWM) |
| | Pitch (x) | PA1 | TIM2 Channel 2 (PWM) |
| **摇杆 (Joystick)** | VRX (Pitch) | PB0 | ADC1 IN8 |
| | VRY (Yaw) | PB1 | ADC1 IN9 |
| | SW (按键) | PA7 | EXTI7 外部中断 |
| **传感器** | MPU6050 SCL | PB8 | I2C1_SCL |
| | MPU6050 SDA | PB9 | I2C1_SDA |
| **调试接口** | USART1 TX | PA9 | 波特率见工程配置 |
| | USART1 RX | PA10 | 用于查看实时坐标与姿态 |

> **供电建议**：舵机在转动时电流较大，建议使用独立 5V 电源为舵机供电，并与 STM32 开发板 **共地 (GND)**，以防 MCU 因电压跌落而复位。

## ⚙️ 软件架构与核心逻辑

### 1. 工作模式 (work_mode)
* **Mode 0 (摇杆模式)**:
    * 读取 `adc_value[0]` (X轴) 和 `adc_value[1]` (Y轴)。
    * **死区处理**：设置 `JOYSTICK_DEADZONE = 150`，有效消除机械中位抖动。
    * **动态校准**：通过内部参考电压通道实时计算 `actual_vdda`，确保在电池电压波动时角度计算依然精准。
* **Mode 1 (MPU6050 模式)**:
    * 读取 MPU6050 解算的欧拉角 (`MM.yaw`, `MM.pitch`)。
    * **反向补偿**：采用公式 `Target = 90.0f - mpu_angle`，抵消载体偏转。
    * **限幅保护**：输出角度严格限制在 `10° ~ 170°`，保护舵机不受机械死点撞击。

### 2. 核心算法模块 (`algorithm.c`)
* `ADCToAngle`: 将 12 位 ADC 值线性映射至 0-180°。
* `MPUToAngle`: 处理传感器数据，包含姿态到舵机行程的逻辑换算。
* `AngleToCCR`: 将目标角度转换为定时器比较寄存器值。标准定义：`500us (0°)` 至 `2500us (180°)`。

### 3. 系统分层设计
* **`main.c`**: 负责硬件初始化、外设启动及主任务分发。
* **`controler.c`**: 封装具体业务逻辑（如任务处理函数及按键中断回调）。
* **`algorithm.c`**: 纯数学计算层，不涉及底层寄存器，易于移植。

## 🚀 快速开始

### 环境要求
* **IDE**: Keil MDK-ARM (v5.30+)
* **配置工具**: STM32CubeMX
* **硬件**: STM32F103 系列开发板 (如 C8T6 / RCT6)

### 编译与烧录
1. 确认 `tim.h` 中 TIM2 的 **Prescaler** 设置为 `71`，**ARR** 设置为 `19999` (以 72MHz 为例)，确保频率为 50Hz (20ms 周期)。
2. 在 Keil 中进行 **Rebuild All**，确认 `0 Error, 0 Warning`。
3. 通过 ST-Link 或 DAP-Link 下载程序。

### 串口调试说明
连接串口助手后，可观察到如下格式数据：
* **摇杆模式**: `Raw_X, Raw_Y, Angle_Yaw, Angle_Pitch`
* **MPU模式**: `Roll, Pitch, Yaw` (角度单位：°)

---

## 📝 注意事项
1. **初始化耗时**：程序启动时 MPU6050 会进行自检与静止校准，此时请勿晃动云台。
2. **volatile 声明**：`work_mode` 变量已使用 `volatile` 修饰，确保中断与主循环间的数据同步。
3. **中断优先级**：建议将定时器中断优先级设为中等，以防阻塞 I2C 通信。
